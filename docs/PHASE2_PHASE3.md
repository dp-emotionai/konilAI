## Фаза 3 (Design & UX)

Цель: единообразный UX, понятные состояния загрузки/ошибок, удобство на мобильных.

### 3.1 Состояния загрузки и ошибок (аудит) — сделано
- Пройти все основные страницы (teacher/student/admin) и убедиться, что везде есть:
  - скелетон или спиннер при загрузке;
  - явное сообщение при ошибке и кнопка «Повторить» где уместно;
  - пустое состояние (нет данных) с коротким пояснением.
- Сделано: teacher dashboard/sessions — кнопка «Обновить», пустое состояние «Нет сессий» + ссылка на создание; teacher sessions — различие «Нет сессий» / «Ничего не найдено» по фильтру; teacher groups — getTeacherGroups бросает при ошибке API, страница: loadError + «Повторить»; teacher reports — русские подписи, ошибка «Не удалось загрузить сессии» + «Повторить»; student dashboard — sessionsError + «Повторить», русские подписи (Join by code → Войти по коду, Consent → Согласие, Student → Студент).

### 3.2 Кнопки и действия
- Единый стиль основных кнопок (primary / outline / ghost).
- Отключённое состояние (disabled) при отправке формы или недоступном действии.
- Подтверждение для деструктивных действий (удаление, завершение сессии и т.п.) при необходимости.

### 3.3 Адаптив и мобильные
- Проверить ключевые сценарии на узкой ширине: дашборды, списки сессий/групп, форма логина/регистрации, страница сессии (preflight + эфир).
- Таблицы: горизонтальный скролл или карточки на малых экранах.
- Навигация и хедер: удобное меню на мобильных.

### 3.4 Тексты и доступность
- Единый тон подписей (русский), заменить оставшиеся англоязычные фразы в UI.
- aria-labels для ключевых кнопок и форм при необходимости.

### Порядок работ
1. Аудит загрузки/ошибок (3.1) — быстрые правки по страницам.
2. Кнопки и подтверждения (3.2).
3. Адаптив ключевых страниц (3.3).
4. Тексты и доступность (3.4).

После завершения 3.1–3.4 Фаза 3 считается выполненной для текущего релиза.
