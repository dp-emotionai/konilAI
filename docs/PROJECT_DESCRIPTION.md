# Полное описание проекта KonilAI (ELAS)

**Документ:** единое описание платформы для презентаций, онбординга и технической документации.

---

## 1. Название и суть

**ELAS (Emotion-Aware Learning Analytics System)** — интеллектуальная система анализа эмоциональной вовлечённости в онлайн-обучении на основе видеопотока с веб-камеры.

**KonilAI** — брендовое название платформы в интерфейсе (лендинг, навбар, маркетинг).

**Ключевая идея:** преподаватель получает в реальном времени метрики вовлечённости, стресса и усталости студентов по видеопотоку с камеры **без записи и хранения видео**, с явным согласием (consent-first) и этичным использованием данных.

## 2. Состав проекта (экосистема)

| Часть | Технологии | Назначение |
|-------|------------|------------|
| **Веб-фронтенд** | Next.js, React, Tailwind CSS | Сайт: лендинг, авторизация, дашборды студента/преподавателя/админа, live-сессии, аналитика. Хостинг: Vercel. |

## 3. Решаемые проблемы

1. **Потеря вовлечённости в онлайне** — на дистанте преподаватель не видит реакцию аудитории; провалы внимания и стресс остаются незамеченными.
2. **Субъективность ручной оценки** — «по лицам» можно оценить только малую группу; при десятках участников объективной обратной связи по эмоциям нет.
3. **Риски на экзаменах и защитах** — важны пики тревожности и динамика состояния по этапам; без инструментов фиксируются только итоговые оценки.
4. **Конфиденциальность** — классический прокторинг записывает и хранит видео; ELAS даёт аналитику **без записи видео** и с явным согласием до анализа.

## 4. Основные возможности

### Для студента

- Регистрация, подтверждение email, вход (логин/пароль).
- **Дашборд:** список сессий, приглашения в группы (принять/отклонить), ссылки «Мои группы» и «Мои сессии».
- **Сессии:** вход по коду/ссылке, просмотр информации о сессии, **согласие на анализ эмоций** (consent) перед включением камеры.
- **Участие в эфире:** включение камеры, передача кадров в ML (только при данном согласии), отображение своего состояния (бейджи), чат сессии.
- **Сводка по эмоциям:** «Моя сводка» — KPI и графики по сохранённым метрикам после сессии; состояния загрузки/ошибки/пусто и кнопка «Повторить».

### Для преподавателя

- **Группы:** создание групп, приглашение студентов (ссылка/код), просмотр участников, исключение/блокировка (с подтверждением).
- **Сессии:** создание сессии (привязка к группе, тип: лекция/экзамен), список сессий (фильтры, поиск), запуск и **завершение сессии** (с подтверждением).
- **Эфир (монитор):** live-подключение по WebSocket, отображение участников и их метрик в реальном времени, баннер при обрыве связи, завершение сессии из эфира.
- **Аналитика сессии:** отчёты по сессии (SessionSummary, Timeline), KPI, табы; экспорт (заготовка под выгрузку).
- **Отчёты:** общий список отчётов, фильтры (все/в эфире/завершённые), поиск.

### Для администратора

- Дашборд и аудит (данные с API), списки пользователей и групп.
- Страницы Model и Storage — демо/локальное состояние (при необходимости позже подключаются к backend).

## 5. Технические особенности

- **Видео не сохраняется:** кадры обрабатываются в реальном времени (1–2 кадра/с), в БД и логах — только агрегированные метрики.
- **Consent-first:** анализ эмоций по лицу не стартует, пока студент не дал явное согласие в интерфейсе.
- **Два сценария ML:** лекция и экзамен — разные веса и пороги в fusion-модуле для итогового состояния и engagement_state.
- **Интеграция:** фронт (Next.js) общается с backend по REST; на странице эфира студента кадры отправляются в `emotion-ml-service` (POST /analyze); сводки сохраняются в backend через ingest API.
- **Единый тон UI:** русский язык интерфейса, единые состояния загрузки/ошибки/пусто и кнопки «Повторить»/«Обновить», подтверждения для деструктивных действий.

## 9. Краткая формулировка для презентации

**KonilAI (ELAS)** — это платформа анализа эмоциональной вовлечённости в онлайн-обучении по видеопотоку с веб-камеры. Система в реальном времени считает вовлечённость, стресс и усталость и показывает их преподавателю в эфире и в отчётах, **не записывая и не храня видео**, с явным согласием студента. В состав входят веб-приложение (Next.js), backend (Node.js, Prisma), ML-сервис (Python) и мобильное приложение (Flutter) с общим API. Проект заполняет нишу между «жёстким» прокторингом и классической LMS-аналитикой, делая упор на этичность и пользу для учебного процесса.
