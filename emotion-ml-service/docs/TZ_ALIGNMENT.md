# Соответствие техническому заданию (ТЗ) ELAS

Краткое сопоставление требований ТЗ и реализации в модели.

## 1. Эмоциональные состояния (ТЗ п. 4.2.1)

| ТЗ (минимальный набор)     | Реализация |
|----------------------------|------------|
| Нейтральное состояние      | `config/tz_emotion_mapping.py`: маппинг в `neutral` |
| Скука / низкая вовлечённость | `boredom` (низкий engagement + Neutral/Sad) |
| Интерес / внимание         | `interest` (Happy/Surprise, высокий engagement) |
| Фрустрация                 | `frustration` (Angry, Disgust) |
| Стресс / тревожность       | `stress_anxiety` (Fear, высокий stress) |
| Уверенность / спокойствие  | `confidence_calm` (высокий engagement, низкий stress, стабильность) |

Модуль **config/tz_emotion_mapping.py** переводит выход CNN (7 эмоций) и метрики (engagement, stress, stability) в эти 6 состояний. В состоянии по каждому кадру доступны поля `tz_state` и `tz_state_label_ru`; для группы — `tz_state_distribution` (доли состояний для отчётов вида «у 68% — скука»).

## 2. Используемые признаки (ТЗ п. 4.2.2)

| ТЗ              | Реализация |
|-----------------|------------|
| Facial landmarks| Пока только bbox (Haar). Landmarks можно добавить (dlib/MediaPipe). |
| Микровыражения  | Не выделяются отдельно; учтены в динамике эмоций по буферу. |
| Динамика эмоций | `temporal/buffer.py`, `analytics/stability.py` (trend, volatility, switch_rate). |

## 3. Аналитика для онлайн-лекций (ТЗ п. 4.3)

| ТЗ | Реализация |
|----|------------|
| График вовлечённости по времени | Данные в `events_log_temporal.csv`, сессия в JSON (engagement по кадрам/сегментам). |
| Точки «провалов внимания» | `analytics/attention_drops.py`: детектор по порогу engagement и длительности; в экспорте сессии — `attention_drops`. |
| Средняя вовлечённость группы | `group_state.avg_engagement`, сессия `averages.engagement`. |
| Сравнение лекций | Несколько сессий: сравнение по JSON/отчётам (start/end, averages, attention_drops). |

## 4. Аналитика для экзаменов и защит (ТЗ п. 4.4)

| ТЗ | Реализация |
|----|------------|
| Средний уровень стресса | `avg_stress` в группе и в сессии. |
| Пиковые моменты тревожности | `stress_peaks` в сессии, `stress_spike` в группе. |
| Эмоциональная стабильность | `stability`, `avg_stability`; в сессии — по сегментам. |
| Уверенность в начале/середине/конце | `analytics/confidence.py`, в сессии — `confidence_segments` (start, middle, end). |
| Таймлайн эмоций | Временные логи + сессия (faces с метриками по времени). |
| Итоговая сводка без баллов | Сессия: averages, changes, patterns, attention_drops, confidence_segments. |

## 5. Участники и имена

По ТЗ групповая аналитика анонимизирована. Реализовано:

- **config/participant_registry.py** — реестр меток участников.
- По умолчанию отображаемый вид: `Participant {face_id}`.
- Опционально можно задать метку: `participant_registry.set_participant_label(face_id, "Иван И.")` или `"Студент 42"` (при согласии/для экзамена).
- В состоянии по лицу есть поле `participant_label`; в экспорте сессии и логах — та же метка (для личной статистики и отчётов преподавателя при необходимости).

Имена/метки не используются для автоматической оценки; только для отображения и экспорта по решению системы/политике конфиденциальности.

## 6. Что можно доработать под ТЗ

- **Facial landmarks** — подключить dlib или MediaPipe для 68/468 точек и использовать в признаках (напрямую или производные).
- **Частота анализа 1–2 кадра/сек** — уже поддерживается (можно троттлить вызов анализа по времени).
- **Видео не сохраняется** — сохраняются только метаданные (логи, сессии); при веб-интеграции не сохранять сырые кадры.
- **Сравнение разных лекций** — на фронте или в отдельном скрипте: загрузка нескольких JSON сессий и сравнение по `averages`, `attention_drops`, `confidence_segments`.

Модель покрывает требования ТЗ по состояниям, аналитике лекций/экзаменов, провалам внимания, уверенности по сегментам и опциональным меткам участников.
